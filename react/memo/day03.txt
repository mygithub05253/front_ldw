react에서의 forEach()
- 새로운 배열을 return 해주지 않음
- 버츄얼 DOM이 변화를 인식하지 못함
- console에 나타내려고 할 때 제외하고는 거의 사용하지 않음

react에서의 map()
- 새로운 배열을 return
- 버츄얼 DOM이 변화를 인식
- 반복문에서 넘겨줄 때 고유한 key 값을 지정해줘야 함

반복문을 사용해서 넘겨주면 각각의 상태를 가지게 됨

훅함수를 활용한 상태 관리
- 앞에 use가 붙은 것은 웬만하면 거의 다 훅함수
- 상태를 관리할 변수, 상태를 바꿀 수 있는 setter 메서드 활용
- 변수와 변수의 메서드를 비구조화 할당을 통해서 활용
- 훅함수를 사용할 시 상태를 바꾸면서 리랜더링 실행
- useState(변수의 초기값); => 변수의 상태 설정
- userRef([]); => DOM 가져오기

DOM 대체 Reference
- onClick : 버튼 등을 눌렀을 때
- onChange : 변화가 발생했을 때

React 주의 사항
- DOM에 직접적으로 접근하는 것을 지양하기 (DOM을 직접 불러오는 행위 지양)
ex) document. 과 관련된 것 사용하지 말기
