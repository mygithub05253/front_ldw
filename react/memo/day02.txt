ReactJS란?
- 복잡한 사용자 인터페이스(UI)를 쉽게 구축하기 위해서 페이스북에서 제작된 자바스크립트 기반의 라이브러리
- 싱글 페이지 애플리케이션(SPA)이나 모바일 애플리케이션 개발에 주로 사용
- 특정 데이터에 변화가 생길 때 매번 변화를 주기 위한 코드를 작성하는 것이 아니라 기존 코드를 없애고 다시 새롭게 랜더링

랜더링(Rendering)
- 화면에 보여주는 것
- 코드가 번역되어 실행된 뒤 화면을 통해 사용자에게 보여짐

초기 랜더링(Mount)
- 가장 처음으로 랜더링되는 것
- render() 함수를 사용하여 veiw를 정의하고 초기 랜더링을 진행

리랜더링(Re-rendering)
- veiw를 업데이트 해야할 때 다시 지웠다가 그려야 하며, 이를 리랜더링이라고 함
- 전체를 모두 지우고 다시 그리는 것은 비효율적이기 때문에 이전 DOM트리와 새로운 DOM트리를 비교
- 비교한 뒤 변경된 요소의 부분만 리랜더링
- 단, 부모가 리랜더링되면 자식도 리랜더링됨
- 반대로 자식이 리랜더링되면, 부모가 리랜더링 가능

컴포넌트(Component)
- React에서 앱을 이루는 최소한의 단위
- 컴포넌트가 모여서 하나의 앱이 되며, 각 컴포넌트는 분리되어 재사용 가능

버츄얼 돔(Virtual DOM)
- 화면보다 가상돔에 먼저 랜더링이 됨 (즉시 값의 변화를 비교)
- 컴포넌트에 변화가 감지되면 Virtual DOM에 랜더링이 됨
- 이전 Virtual DOM과 현재 Virtual DOM을 비교하여 이로 인해 필요한 부분을 리랜더링

결론
1) React는 화면만 구성하는 라이브러리 => 서버가 아님
2) 자바스크립트가 들어가기 때문에 통신이 가능
3) 따라서 프론트엔드이다.
4) 상태는 state라고 하며 비동기이다.

React의 연산
- return() 안에서는 html이기 때문에 연산이 안됨
- 연산을 하기 위해서는 {} 중괄호로 묶어줘야 함

JSX는 if문을 사용할 수 없기 때문에 삼항 연산자를 지원
- return() 안에서만 if문 사용 불가능
- 조건식 ? 참일 때 랜더링할 JSX : 거짓일 때 랜더링할 JSX
- 조건식 && 참일 때 랜더링할 JSX, 거짓이면 아무것도 출력하지 않음
- && 연산자의 앞의 조건식이 false이면 뒤를 실행하지 않으며, false라는 값은 랜더링되지 않음
